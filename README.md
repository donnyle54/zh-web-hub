# zh-web-hub

## 项目简介

zh-web-hub 是一个用于归档和发布多个独立 HTML 页面的仓库。这些页面可能包含工具、信息展示、实验性设计或其他网页内容，每个页面均以独立的 HTML 文件形式存在，不依赖于特定的域名或外部服务。

本仓库的主要目的是集中管理和长期保存这些页面，方便用户通过 GitHub Pages 或其他静态托管服务直接访问。

## 目录说明

```
zh-web-hub/
├── index.html          # 仓库首页，通常用于导航或页面列表
├── pages/              # 存放所有独立 HTML 页面
│   ├── page1.html
│   ├── page2.html
│   └── ...
├── assets/             # 公共资源文件（如 CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- **pages/**：所有归档的 HTML 页面均放置在此目录下，每个文件代表一个独立页面。
- **assets/**：存放页面共用的样式、脚本或图片资源，便于维护和复用。
- **index.html**：根目录下的入口文件，可用于展示页面列表或提供快速导航。

## 页面归档说明

- 每个页面文件应保持自包含性，尽量将样式和脚本内联或引用自 `assets/` 目录下的资源。
- 页面内容应遵守相关法律法规，不包含恶意代码、侵权内容或不当信息。
- 开发者可自由添加新页面，但建议在 `pages/` 下按功能或主题进行组织（例如创建子目录）。

## 维护说明

- **添加新页面**：在 `pages/` 目录下创建新的 HTML 文件，并确保其链接或路径在 `index.html` 中有所体现（可选）。
- **更新资源**：修改 `assets/` 下的文件时，请注意检查所有引用该资源的页面是否兼容。
- **删除页面**：移除对应 HTML 文件，并更新 `index.html` 中的导航链接（如有）。
- 建议在提交更改前进行本地测试，确保页面在静态环境下可正常显示。
